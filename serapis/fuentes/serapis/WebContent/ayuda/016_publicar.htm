Publicar

Esta opción permite publicar los registros o “subir” los archivos desde el equipo a SERAPIS, para lograr esto SERAPIS presenta una página con un formulario que permite publicar el documento, este formulario esta compuesto por:

 
 

Proceso. Permite la selección del proceso al cual pertenece el registro que se subirá.
Tipo Documento. Establece el tipo de documento que se subirá.
Nombre. Permite la selección del nombre del documento, los nombres de los documentos aparecerán dependiendo de la selección de las opciones realizadas para Proceso y Tipo de Documento.
Versión Registro. Permite que se ingrese la versión del registro, este es un número que va desde 01 a 99. Siempre se deben ingresar los dos dígitos.
Cliente. Cliente al que se hace referencia el registro. Si es propio de la empresa, se debe seleccionar esta.
Adicional. Adicional del registro. Este adicional formará parte del nombre del registro.
Número Correlativo. Correlativo de registro correspondiente a la versión.
Descripción. Es un comentario que se puede agregar al documento y que sirve de descripción para este.
Comentario Adicional. Texto adicional que se asocia al registro. Es optativo.
Archivo. Es la ubicación del archivo. Con el botón Examinar se puede ubicar el archivo que se subirá a SERAPIS.
Una vez que se ingresan todos los campos anteriores, por medio del botón Subir se agregará el registro al Sistema, al subir se realizarán una serie de validaciones, si estas se cumplen en su totalidad el documento estará disponible. Si existiese un documento que corresponde al Proceso, Tipo de documento, Nombre y Versión Registro, Cliente y Correlativo se avisará que el documento no es posible subirlo ya que existe uno previo con las mismas características.
Una vez que el archivo ya se cargue, se mostrará una página especificando que el archivo se copio directamente al Sistema.

 

Al igual que los registros aprobados, la publicación es posible realizarla de dos formas, la primera es la Standard y que se asocia directamente el proceso, pero para asociar un registro a la dupla Cliente/Proyecto, la publicación se realiza por medio de la opción Publicar – Proyecto.
El formulario de publicación, tiene las mismas características que el que se muestra en la Figura, pero adicionalmente se solicita el ingreso del proyecto asociado al cliente, en la se muestra el formulario de publicación de registros para Clientes/Proyectos.

 
 

En este caso el ingreso del proyecto es obligatorio y al guardar el registro se considera para ver la no existencia de versiones anteriores.

Asignación de WorkFlow


Todos los registros que es posible que maneje SERAPIS están previamente definidos, uno de los componentes de la definición corresponde a un WorkFlow, el que permite realizar la autorización del documento en “n” pasos donde existen diferentes usuarios involucrados. Si el registro que se publica tiene asociado a un WorkFlow, el registro queda en estado En Flujo lo que significa que el registro esta en espera de aprobación, cuando este es el caso, SERAPIS informa que el archivo se copio exitosamente al sistema, pero que ha ingresado a un flujo de aprobación. Un ejemplo del aviso entregado de aprecia en la Figura.

 
 

Como se aprecia, el aviso incluye el nombre del WorkFlow y el encargado de realizar el primer paso del WorkFlow, este usuario es notificado de manera electrónica (E-Mail) para que realice la acción que corresponda para realizar la aprobación final del documento.
