Inicio de Sesión

El sistema solicitará inicio de sesión cuando alguna de las opciones lo requiera. Cuando se ha iniciado sesión y por un largo tiempo (20 minutos) no se la usado el sistema, SERAPIS solicitará nuevamente que se ingrese la información de autenticación. Para iniciar sesión debe ingresar el usuario y clave. Si se he ingresado por primera vez su clave es el mismo usuario, el sistema solicitará que cambie su clave y posteriormente deberá iniciar sesión nuevamente.

En la etapa de autenticación podrían ocurrir los eventos que se describen a continuación:

  • Usuario no existe. Ocurre cuando el identificador de usuario no es encontrado en la tabla de usuarios del sistema.
  • Clave incorrecta. Indica que la clave del usuario no es válida, este chequeo se realiza contra la clave registrada del usuario
  • Usuario inactivo. Ocurre cuando el Usuario que intenta ingresar se encuentra deshabilitado de ingresar al Sitio. Si este es el caso, debe informar al administrador para autorizar su ingreso
 

Al momento de iniciar sesión es posible que cambie su clave. El sistema solicitará cambio de clave cuando el usuario no haya ingresado nunca al sistema o cuando el último cambio de clave haya ocurrido hace más de 30 días. Para cambiar la clave debe ingresar la clave actual e ingresar dos veces su nueva clave. Al realizar el cambio, deberá ingresar nuevamente la información de autenticación.

Los eventos que pueden ocurrir en esta página son:

  • Clave Incorrecta. Ocurre cuando la clave actual ingresada no corresponde a la que se encuentra en los registros del sistema.
  • Usuario no existe. Ocurre cuando hay un problema de conexión a la base de datos.
  • Clave no válida. Indica que la clave nueva no cumple con los requisitos de seguridad de clave que contempla el sistema.
  • Usuario Inactivo. Informe al administrador para autorizar su ingreso. Ocurre cuando el Usuario que intenta ingresar se encuentra deshabilitado de ingresar al Sitio. Si este es el caso, debe comunicarse con el administrador del sistema e indicar que realice la habilitación del usuario.
  • Sesión Finalizada. Por seguridad ingrese su autenticación nuevamente. Ocurre después de 20 minutos en que no ha habido actividad por parte del usuario en el sistema.
  • Problemas con su Licencia. Contacte a su proveedor. Problemas con la licencia por motivos de vencimiento.
 

Notas:

  • Encriptación de Claves. Las claves de los usuarios se almacenarán en todo momento de manera encriptada por motivos de seguridad.
  • Requisitos de seguridad de clave. Para dar un nivel mayor de seguridad se establecen reglas para el cambio de claves. Estas reglas se describen a continuación.
  • Largo. La clave tendrá un largo máximo de 10 caracteres.
  • Caracteres válidos. Se consideran válidas las claves que contengan caracteres que se encuentren entre A-Z, a-z, 0-9, signos de puntuación (punto, coma, punto y coma), y los caracteres +,-,*,/,>,<,=..
