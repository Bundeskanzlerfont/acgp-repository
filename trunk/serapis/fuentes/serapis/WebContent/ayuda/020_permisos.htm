Permisos

El almacenamiento de registros en ocasiones involucra información sensible y estratégica que es conveniente manejar de forma reservada, es por esto que este módulo del sistema contempla un sistema de autorizaciones que va más allá de la simple definición de usuarios y encargados de procesos, sino que se pueden dar detalles a un nivel más atómico. Los permisos se otorgan de dos formas, para procesos y proyectos.

 
Permisos por Proceso


Un usuario definido como líder de proceso o backup habilitado es el único que puede dar permisos a los registros de su proceso, los permisos pueden ser dados a un usuario en particular o grupo de usuarios. Los usuarios que tengan permisos sólo podrán ver los registros, y los que no lo tengan, el sistema informará que no poseen los privilegios al momento de solicitar la documentación.

Nota.El líder debe darse permisos a si mismo para ver los registros de su área si desea ver sus registros.

Al seleccionar la opción de permisos por proceso, el sistema desplegará una lista con las autorizaciones asignadas ordenadas por proceso, bajo la lista existe un botón Nuevo que permite el ingreso de un nuevo permiso.

 

Cada acceso posee un link en el nombre para ver el detalle del permiso asignado.

 
Editar Permiso

La edición permite modificar o eliminar el permiso asignado. Como se mencionó es posible dar permisos a un usuario o grupo de usuarios, estos grupos de usuarios denominados perfiles están creados previamente. La edición de cada permiso contempla:

 

Proceso. Proceso al que se asignó el permiso Tipo. Si es a un Usuario o Perfil. Usuario. Si tipo es Usuario, aparece el nombre del usuario al que se dio permiso, si es Perfil, el nombre del perfil. Acceso. Acceso asignado, el que puede ser:

  • Control Total. Tendrá acceso total a los registros del proceso.
  • Escritura. Podrá descargar los documentos y modificar su descripción.
  • Sólo lectura. Podrá descargar los documentos.
  • Eliminación. El usuario podrá eliminar registros.
  • Sin Acceso. No puede acceder a los documentos del proceso.

Al presionar sobre el botón Modificar se modificará el acceso, para eliminar se debe presionar sobre el botón Eliminar.

 
Nuevo Permiso

Permite crear un nuevo acceso para el proceso, el formulario que permite crear el acceso consta de:

 
 

Proceso. Proceso al que se asignó el permiso, el listado de procesos corresponde a los que el usuario actual es lider o backup habilitado.
Perfil. Especifica el grupo de usuarios a los que se dará el mismo permiso.
Usuario. Especifica el usuario al que se dará el permiso. Si se ha seleccionado un perfil, el sistema no considera el usuario, por lo tanto, es válido sólo si no hay seleccionado un perfil.
Acceso. Acceso asignado, el que puede ser:

  • Control Total. Tendrá acceso total a los registros del proceso.
  • Escritura. Podrá descargar los documentos y modificar su descripción.
  • Sólo lectura. Podrá descargar los documentos (por defecto).
  • Eliminación. El usuario podrá eliminar registros.
  • Sin Acceso. No puede acceder a los documentos del proceso.

Para grabar el permiso, se debe presionar sobre el botón Aceptar.

 
Permisos por Proyecto

Permite asignar los permisos de los registros asociados a Clientes/Proyectos, la opción permite buscar el permiso o permisos por proyecto, cliente o jefe de proyecto.
Esta opción de seguridad esta restringido a los usuarios que son jefes de proyecto o backup habilitado de los proyectos de los clientes.

 
 

Se debe buscar necesariamente el proyecto al que se asignará el permiso, la búsqueda se puede realizar por Proyecto, Cliente o Jefe de Proyecto (sólo 1 de los tres). Al buscar el sistema desplegará los proyectos asociados a la búsqueda.

 
 

Para cada proyecto se desplegarán los permisos asociados, al seleccionar el permiso este se podrá editar, al final de la lista de permisos además se podrán crear nuevos permisos por medio del botón Nuevo.

La edición permite modificar o eliminar el permiso asignado. Como se mencionó es posible dar permisos a un usuario o grupo de usuarios, estos grupos de usuarios denominados perfiles están creados previamente. La edición de cada permiso contempla:

 
 

Cliente. Cliente al que esta asociado el proyecto. Proyecto. Proyecto asociado al cliente al que se da permiso. Usuario. Si tipo es Usuario, aparece el nombre del usuario al que se dio permiso, si es Perfil, el nombre del perfil. Acceso. Acceso asignado, el que puede ser:

  • Control Total. Tendrá acceso total a los registros del proceso.

  • Eliminación. El usuario podrá eliminar registros.
  • Sin Acceso. No puede acceder a los documentos del proceso.

Al presionar sobre el botón Modificar se modificará el acceso, para eliminar se debe presionar sobre el botón Eliminar.

Cliente. Cliente al que pertenece el proyecto. Proyecto. Proyecto asociado al cliente al que se da autorización. Perfil. Especifica el grupo de usuarios a los que se dará el mismo permiso. Usuario. Especifica el usuario al que se dará el permiso. Si se ha seleccionado un perfil, el sistema no considera el usuario, por lo tanto, es válido sólo si no hay seleccionado un perfil. Acceso. Acceso asignado, el que puede ser:
